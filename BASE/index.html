bSunDAI Guide
Autonomous ETH-Backed Stable Asset on Base · V7

What is bSunDAI?

bSunDAI is a USD-pegged stable asset backed by ETH. You lock ETH as collateral and borrow bSunDAI against it. Each bSunDAI targets $1 USD. The peg is enforced by over-collateralization, a 0.5% annual stability fee, and a redemption mechanism where bSunDAI can always be redeemed for $1 worth of ETH from any vault.

There are no admin keys. No governance. No pausing. The vault, token, and oracle are immutable after the one-time setup. The system runs autonomously on Base Chain.

Opening a Vault

Option A (Auto): Enter an ETH amount and click "1-Click Auto Borrow". Deposits ETH and mints bSunDAI at 155% CR in one transaction.

Option B (Manual): Deposit ETH first → then Mint bSunDAI separately. Useful when you want a specific CR or only need the ETH deposit without borrowing.

Minimum amounts: 0.0001 ETH per action. Minimum redemption: 100 bSunDAI.

The Oracle (V7 Dual-Price System)

V7 runs two independent price tracks:

Committed Price — Used for all minting and withdrawal safety checks. Accepts small moves (≤2% down, ≤5% up) instantly. Larger moves start a confirmation period (4h down, 30min up) to prevent liquidation cascades during flash crashes.

Chainlink Warning Track — Monitors whether live Chainlink is diverging from committed. If Chainlink stays 3%+ below committed for 30 continuous minutes, liquidations switch to live Chainlink price. This closes the bad-debt window that conservative committed stepping creates during a genuine crash. Flash crashes recover in minutes — the 30-min threshold filters them out.

The oracle is a public utility — anyone can call refreshPrice() to keep it up to date.

Liquidations

Any wallet holding bSunDAI can liquidate a vault below 110% CR. You repay some (or all) of the vault's debt and receive proportional ETH collateral plus a bonus.

Inverted Dutch auction: Bonus starts at 10% immediately and decays to 2% over 3 hours. First mover wins the highest bonus — the vault cannot recover without action. This is reversed from V6 (where the bonus grew over time), preventing bots from waiting and letting vaults accumulate bad debt.

Go to the Liquidations page to scan all vaults and act on eligible ones.

Redemptions

Any wallet can redeem bSunDAI for ETH at the current oracle price. 1 bSunDAI = $1 USD worth of ETH, always. Redemptions are available against any vault with sufficient debt — there is no minimum CR requirement to be a redemption target.

You receive: bSunDAI_amount × $1 / ETH_price ETH, minus a 0.5% fee. The 0.5% fee stays with the vault owner as compensation. Minimum: 100 bSunDAI per redemption.

This creates a hard peg floor: if bSunDAI ever trades below $1 on the open market, arbitrageurs buy cheap and redeem at full $1 value — closing the gap instantly.

Stability Fee & Surplus Buffer

A 0.5% annual stability fee accrues continuously to your debt. Example: 1,000 bSunDAI debt adds ~5 bSunDAI per year. This fee accumulates in the Surplus Buffer — protocol equity held on-chain that absorbs future liquidation losses before bad debt propagates.

If a liquidation leaves uncovered debt, it goes into badDebtAccumulated. The surplus buffer automatically nets against it via reconcile(). System Equity = Surplus Buffer − Bad Debt.

Debt Ceiling

The vault has an immutable debt ceiling set at deploy time. Minting reverts if the new total debt would exceed it. The ceiling limits protocol-level risk — it cannot be increased without deploying a new vault. The System State panel shows current utilization.

Emergency Functions

Emergency Unlock: If you have zero debt and your last deposit was 30+ days ago, you can call emergencyUnlock() to recover collateral regardless of oracle state. The button appears automatically when conditions are met.

Emergency Repay / Withdraw: Available only when the oracle has been completely dead for 7+ days. Allows orderly exit without oracle price. Not relevant under normal conditions.

Base Chain Setup

Chain ID: 8453. RPC: https://mainnet.base.org. The page will prompt MetaMask to switch to Base automatically if you're on a different chain.

You interact with ETH directly — the vault wraps to WETH internally. You never need to wrap manually. Withdrawals automatically unwrap back to ETH.
